# socratic-dialogue

> 把任何文章 / 产品 / 观点 / 道理,写成一场苏格拉底式对话 —— 一个信徒兴奋陈词,苏格拉底层层追问,最后反转留白,不替读者下结论,让读者跟着信徒一起被问醒。

一句『把这个写成苏格拉底对话』 → 一篇有"较量感"的对话体文章(信徒有牙、苏格拉底不认错、aporia 收尾),可选配网页排版。

## 它是什么

一个 Claude Code / Claude skill。明说要"对话体 / 苏格拉底式"才触发,例如:

- "把这篇文章写成苏格拉底对话"
- "用对话体讲讲『AI 是产品之王』这个判断"
- "苏格拉底式拆解一下『一切都会被颠覆』"
- "仿柏拉图对话录,让苏格拉底和某人辩一辩 X"

触发后产出:

- **一篇完整对话体文章** —— 信徒陈词 → 苏格拉底层层追问 → 反转 → aporia 收尾
- **三角色编排** —— 信徒(读者化身,真诚可信、有牙)/ 苏格拉底(只问不答、装无知)/ 旁白"我"(可选)
- **可选网页排版** —— 双色区分两人、羊皮纸调、手机自适应、金句卡 + 参考文献

不是"AI 替你讲道理"的议论文,是**让读者站在场边看两人较劲、自己卸下防备被问醒**的对话。

## 为什么用对话体(而不是普通议论文)

| 普通议论文 / 通用 AI 写作 | socratic-dialogue |
|---|---|
| "我对你讲道理",读者抬杠、划走 | "你看这两人较劲",读者卸防备 |
| 替读者下结论,说教感重 | aporia 收尾,不下结论,留白回味 |
| 信徒当提线木偶,句句认输 | 铁律 1:信徒必须有牙,赢一两回合 |
| 苏格拉底"对不起我错了" | 铁律 2:苏格拉底绝不认错,靠三招化解 |
| 一口气甩排比 / 抒情独白 | 铁律 3:不许独白,排比拆成问答 |
| "AI 生不出 X"全称句,一反例就打脸 | 铁律 4:reframe 成"AI 生出的恰是 X 的反面" |
| 堆古文充希腊感,假、端着 | 铁律 5:内容平白,只在连接转折处用译本腔 |
| cosplay 穿越戏,廉价 | 铁律 6:结尾揭穿是虚构 = 自我审判,高级 |
| 满篇破折号当"郑重提示器" | 铁律 7:去 AI 腔,破折号全文 ≤3 个 |

整套招法来自**柏拉图对话录真功夫**:先逼定义(elenchus)、用反例击穿、先骗同意再合拢、越好的对话信徒越倔。

## 用法

**触发词**(必须明说要对话体,普通写文章 / 润色不触发):

- "把 X 写成苏格拉底对话"
- "用对话体写 / 讲 X"
- "苏格拉底式拆解 X"
- "仿柏拉图对话录讲讲这个观点"
- "让苏格拉底和某人辩一辩 X"

**流程**:

```
用户给文章/产品/道理
        │
        ▼
① 抓"时髦判断" ── 核心主张交给信徒去信
        │
        ▼
② 找反方弹药 ── 最强 3-4 个反驳 → 各翻译成苏格拉底一问
        │
        ▼
③ 排弧线 ── 陈词 → 追问(中间信徒赢 1-2 回合) → 反转 → aporia
        │
        ▼
④ 写 ── 遵守七条铁律
        │
        ▼
⑤ 配引用 ── 真人真事列参考文献,拿不准降精度
        │
        ▼
⑥ 问排版 ── "要不要配网页 HTML?" → 套 dialogue-template.html
```

## 核心流程速览

1. **抓时髦判断**:用户给的文章 / 产品 / 道理,核心主张是哪一句?这句交给信徒去信。
2. **找反方弹药**:这个判断最强的三四个反驳是什么?(必要时派 subagent 从哲学 / 产业 / 反方角度挖。)每个反驳翻译成苏格拉底的一个提问。
3. **排弧线**:信徒兴奋陈词 → 层层追问(中间让信徒赢一两回合)→ 反转 → aporia 收尾。
4. **写**:严守七条铁律 —— 信徒有牙、苏格拉底不认错、不许独白、结论 reframe、希腊感靠结构词、旁白揭穿是虚构、去 AI 腔。
5. **配引用**:用到真人真事真概念,文末列参考文献;经典文本只标篇目,拿不准的事实降精度或贴来源。
6. **问排版**:写完问用户要不要配网页 HTML,要的话套 `assets/dialogue-template.html`。

## 七条铁律(成败所在)

| # | 铁律 | 一句话 |
|---|---|---|
| 1 | 信徒必须有牙 | 头号生死线 —— 信徒反咬、赢一两回合,不当提线木偶 |
| 2 | 苏格拉底绝不认错 | 靠"你驳的不是我" / 顺势转向 / 反讽+aporia 化解,不道歉 |
| 3 | 不许独白 | 排比拆成问答,抒情让给信徒自己说 |
| 4 | 结论 reframe | 别用全称句"AI 生不出 X",改"AI 生出的恰是 X 的反面" |
| 5 | 希腊感靠结构词 | 内容平白,只在连接转折处用译本腔,别堆古文 |
| 6 | 旁白弧线 | 开场装真,结尾揭穿是虚构 = 自我审判 |
| 7 | 去 AI 腔 | 破折号 ≤3 个,不当"郑重提示器";短句口语 |

## 自定义

1. **角色名**:信徒默认半古典名(斐洛 / 格劳孔 / 斐德),改 SKILL.md「三个角色」段。
2. **要不要旁白**:旁白"我"可选,不要就删开场 / 结尾场景。
3. **反转强度**:中间让信徒赢几回合,在「对话的弧线」里调。
4. **排版样式**:双色 / 羊皮纸调改 `assets/dialogue-template.html` 的 CSS 变量。
5. **参考文献精度**:经典文本只标篇目;新闻 / 观点拿不准就降精度或贴来源,避免造塌房点。

## 已知限制

- **不主动触发**:必须明说"对话体 / 苏格拉底式",普通写文章 / 润色不触发(防误触)。
- **信徒有牙靠模型自觉**:铁律 1 是软约束,信徒退化成提线木偶时需人工顶回去重写。
- **真人真事需核查**:引用真实新闻 / 观点可能不准,文末参考文献要自己复核,别造塌房点。
- **网页排版为可选步**:HTML 排版要用户明确要才出,默认只出正文。
- **长度不固定**:对话长短取决于反方弹药数量与回合数,不保证字数。

## 贡献

欢迎 PR / issue,重点方向:

- **更多成品范例**:不同题材(产品 / 哲学 / 时事)各加一篇带拆解的 worked-example。
- **铁律打磨**:发现信徒退化、苏格拉底认错等失败模式,补进铁律或落地步骤。
- **排版模板**:深色模式、更多金句卡样式、打印 / PDF 适配。
- **反 AI 腔清单**:破折号之外的 AI 口头禅,持续补充进铁律 7。
